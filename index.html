Artifact

The original artifact used for this review and enhancement is a mobile application that I previously developed in my CS 360: Mobile Architecture and Programming course. This project is a mobile-only inventory management application that allows users to manually add items to a list and perform full CRUD (Create, Read, Update, Delete) operations. The application includes user authentication features such as login and registration, with all user and item data stored locally in an SQLite database on the device.

In the accompanying video, I conduct a detailed code review of the original, unenhanced project, highlighting both its strengths and areas for improvement identified during my learning process. The demonstration also showcases the app's core functionality and provides an overview of how the underlying database supports its operations.

Enhancements

Enhancement One - Software Design and Engineering

For this enhancement, I transformed an existing Java-based mobile application into a full-stack web application using HTML, CSS, and JavaScript. The goal was to demonstrate my ability to migrate software across platforms while maintaining code reusability, scalability, and adaptability.

I implemented secure user authentication through a custom auth.js file using JSON Web Tokens (JWT), showcasing my understanding of secure backend development and my ability to integrate both front-end and back-end functionality. During development, I overcame challenges in syncing front-end and back-end components, strengthening my skills in debugging, problem-solving, and iterative development. This enhancement laid the groundwork for future database and algorithm integration, representing a key step in my growth as a full-stack developer. This process provided valuable insight into the complexities of software migration, illustrating that transferring a program between languages and environments is not a direct one-to-one conversion. It demands thoughtful restructuring, especially on the backend, to maintain consistent performance and reliability. Despite time constraints and technical challenges, this experience deepened my understanding of software design principles, project planning, and iterative development methodologies.

In relation to my personal goals and the course outcomes, I believe this enhancement successfully achieved its intended objectives. My primary goal was to transform a mobile-only application into a functional full-stack web prototype, and through this effort, I demonstrated proficiency in software design, engineering practices, and secure application development.

Enhancement Two - Algorithms and Data Structures

For my second enhancement, I expanded the functionality of the artifact by introducing array and map data structures to manage both items and users. Although a database has not yet been integrated, I implemented in-memory data handling to simulate backend storage and management. Additionally, I incorporated a new search feature, something the original artifact lacked, allowing for efficient data retrieval and improved user experience.

This enhancement represents the most robust portion of the project. I designed a simple yet effective data structure with integrated search logic to manage users and items. Using my app.js file, I established a modular Express backend capable of proper routing and middleware handling, while the api.js file centralized API calls with token-based access control and error handling. This work highlights my growing proficiency in implementing algorithms and data structures within a full-stack architecture. Throughout this process, I encountered challenges related to frontend and backend integration as well as time constraints. These obstacles deepened my understanding of software design principles and reinforced my ability to adapt solutions to real-world development issues.

In relation to the course outcomes, this enhancement demonstrates my ability to apply data structures and algorithms effectively within a full-stack web prototype. I successfully met my objectives by creating a functional and scalable data system that brings the project closer to becoming a fully operational full-stack application.

Enhancement Three - Databases

In my final enhancement, I transitioned the project from an in-memory data structure to a more robust MongoDB database, greatly improving scalability and persistence. I implemented a role-based access control system that introduced distinct user and admin roles. Admins now have the ability to edit and manage users and items, while each item entry includes user and timestamp tracking for accountability. Additionally, I developed an admin panel that allows administrators to remove users or items as needed.

To enhance functionality, I built a live search system with improved performance and logic for automatically merging duplicate items by updating their quantities. This optimization increased both efficiency and the overall user experience. I maintained a clear separation of concerns across core files—login.js, itemlist.js, auth.js, constants.js, and api.js—ensuring organized, modular design and effective use of local storage. I strengthened security by validating user inputs, implementing password hashing, and avoiding any hardcoded security risks within the application.

During development, I encountered challenges with data parsing and file mismatches, which at times resulted in NaN (Not a Number) errors when handling user and item data. Through repeated testing and debugging, I successfully identified and resolved these issues, improving overall system reliability.

The primary goal of this enhancement was to complete the migration from a Java-based mobile application to a fully functional full-stack web application. In alignment with the course outcomes, this enhancement demonstrates my ability to design and implement secure, scalable, and maintainable systems. It highlights my initiative and growth in applying software engineering principles—particularly in areas of backend integration, data management, and security—culminating in a comprehensive, production-ready full-stack solution.
